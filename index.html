What is WebDAV?

"WebDAV stands for "Web-based Distributed Authoring and Versioning". It is a set of extensions to the HTTP protocol which allows users to collaboratively edit and manage files on remote web servers." Taken from http://www.webdav.org/

In simple terms, WebDAV is a web protocol that allows a webpage to act as a server that multiple people can access at one time.

What can WebDAV do?

WebDAV lets users set up private servers that can be accessed from anywhere. They can upload, download, rename, move and delete files freely, as if the flie was local on their own computer. These servers can be protected with usernames and password settings which means that only allowed users can access the server.

Why should I use WebDAV?

WebDAV is useful for geographically disperse teams because it means that they can all access the files they need and know that only people with permission have access to those files. In addition, WebDAV has features in place that protects the files from being overwritten by two different users. The feature, Locking, means that files are checked out to a user for a period of time where the file can only be used by them without fear that their coworker or peer is editing the same file. The user is check out the file for a configurable amount of time and can be checked in anytime between then and the user completing their use of the file.
WebDAV also supports XML which means that the user can store, set and delete metadata on the server. A use of this would be a list of all the authors on the project, a contact list for the users, or a log of who edited what file when.
